# Module 4 · Evaluation

**Part 2 — A Single Capable Agent** · 5 lessons + lab · ~6 hours total

<LearningObjectives>
- Explain why evaluation is the hardest part of agent engineering — non-determinism, no ground truth, multi-step trajectories
- Build eval datasets that include happy path, adversarial, edge case, and regression cases
- Apply and compare three scoring strategies: exact match, rubric scoring, and trajectory analysis
- Use LLM-as-judge correctly and identify its failure modes (position bias, length bias, sycophancy)
- Write regression tests for non-deterministic systems and integrate them into a CI pipeline
</LearningObjectives>

## What you'll build

An eval harness for the Module 3 research agent. By the end of the lab you'll have: a suite of 15+ eval cases spanning all four categories, a response-level scorer, a step-level scorer that validates tool call sequences, an LLM-as-judge for qualitative criteria, and a regression runner that flags when a change breaks a previously passing case. This harness carries forward into every module that follows.

## Why evaluation comes this early

You could ship an agent without an eval suite. Teams do it all the time. They discover the failure modes later — in production, under user load, after a prompt change silently broke something. This module is placed before any workflow or multi-agent work deliberately: **if you can't measure your agent, you can't safely change it.** The eval harness you build here is the foundation everything else is built on.
