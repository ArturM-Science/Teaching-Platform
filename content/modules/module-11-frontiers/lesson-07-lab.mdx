# Lab - Evaluate a Frontier Agent Capability

<LabBrief
  title="Evaluate a Frontier Agent Capability"
  time="2-3 hours"
>

#### Objective

Prototype one frontier agent capability in a constrained setting and produce a production-readiness memo. Your goal is not to prove the capability is amazing. Your goal is to discover what it can do, where it fails, and what controls would be required before using it with real users.

#### Choose one track

- Browser agent: automate a low-risk web workflow and verify each step
- Multimodal agent: inspect a document, screenshot, chart, or image set with cited evidence
- Protocol/tool agent: connect to a small tool server and evaluate tool discovery and permissions
- Long-horizon agent: run a multi-step task with checkpoints, budgets, and pause/resume state

#### Deliverables

- `prototype/` - minimal working prototype or detailed runnable notebook
- `readiness_memo.md` - completed readiness framework with scores and decision
- `failure_log.md` - at least 10 trial runs with success/failure notes
- `trace_sample.json` - one representative run log with observations, actions, and verification
- `control_plan.md` - permission boundaries, approval gates, budgets, and escalation triggers

</LabBrief>

## Step 1 - Define the capability

Write a one-paragraph task definition:

```md
Capability:
Target user:
Job to be done:
Why this is not solved by a simpler tool:
Irreversible or high-impact actions:
```

Keep the prototype small. A browser agent that fills a harmless local form is better than an ambitious agent that cannot be verified.

## Step 2 - Build the smallest realistic prototype

Your prototype must include at least three observable steps. For each step, log goal, observation, action, expected result, actual result, and status.

## Step 3 - Run failure trials

Run at least 10 trials:

- 4 happy-path cases
- 2 ambiguous cases
- 2 missing-context cases
- 2 adversarial or misleading cases

Log failures plainly. A useful failure log is a successful lab.

## Step 4 - Score readiness

Score the prototype from 1 to 4 across task fit, reliability, observability, control, evaluation, UX, and maintainability. Then decide whether it is production-ready, pilot-only, experimental, or rejected for now.

## Step 5 - Write the control plan

Your control plan must include permission scope, approval gates, autonomy budget, observability events, cost budget, escalation condition, and rollback or cancellation path.
