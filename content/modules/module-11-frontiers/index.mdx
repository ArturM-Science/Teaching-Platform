# Module 11 - Frontiers

**Part 5 - Frontiers & Capstone** - 6 lessons + lab - ~6.5 hours total

<LearningObjectives>
- Separate durable agent-system ideas from short-lived frontier hype
- Explain when computer-use, browser, and multimodal agents are useful and where they are brittle
- Design reliability checks for long-horizon autonomous runs
- Evaluate agent protocols, tool discovery, and shared context boundaries without ignoring security
- Apply a readiness framework before adopting an emerging agent capability in production
- Build and critique a small frontier-agent prototype
</LearningObjectives>

## Why this module exists

The frontier moves quickly. New models, protocols, and agent products can look production-ready in a demo while still failing under ordinary operational pressure. This module gives you a practical evaluation lens: what changed, what became possible, what remains fragile, and what you would need before trusting it with users.

The point is not to predict the future. The point is to become difficult to fool.

## What you will build

In the lab you will choose one frontier capability, prototype it in a constrained setting, and write a production-readiness memo. The prototype can be a browser agent, a multimodal document inspector, a long-horizon task runner, or an MCP-style tool integration. The memo is as important as the code: you must explain where it fails, how you observed it, and what would block production use.
