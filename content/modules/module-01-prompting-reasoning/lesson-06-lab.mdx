# Lab · Self-Correcting Code Agent

<LabBrief title="Self-Correcting Code Agent" time="2–3 hours">

#### Objective

Build an agent that takes a coding task and a test suite, generates Python code, runs the tests, reads the failure output, and retries until the tests pass or the retry limit is reached. The agent's reasoning trace must be visible and loggable.

#### Setup

```bash
pip install openai anthropic pydantic pytest
export OPENAI_API_KEY="sk-..."  # or use Anthropic
```

#### Deliverables

- `agent.py` — the main agent loop with generate → test → retry cycle
- `validator.py` — subprocess test runner that returns structured pass/fail output
- `prompts.py` — system and user prompt templates (separated from logic)
- `models.py` — Pydantic models for all structured outputs
- `README.md` — one paragraph explaining where the loop can get stuck and how you bounded it

#### Hard constraints

- Maximum 3 retry attempts — not negotiable
- The loop must stop if the same error appears on consecutive attempts (convergence check)
- Every retry must include the full error output from the failed tests
- The agent's reasoning trace (attempt number, error, retry message) must be logged
- No unhandled exceptions — all failures must return a structured error

</LabBrief>

```python
def run_agent(task: str, test_suite: str) -> list[AgentTrace]:
    messages = [
        {"role": "system", "content": SYSTEM_PROMPT},
        {"role": "user",   "content": build_user_prompt(task, test_suite)}
    ]
    traces = []
    last_error = None

    for attempt in range(1, MAX_RETRIES + 1):
        log.info(f"Attempt {attempt}/{MAX_RETRIES}")

        code_output = generate_code(messages)
        messages.append({"role": "assistant", "content": code_output.model_dump_json()})

        result = run_tests(code_output.code, test_suite)

        status = "success" if result.passed else (
            "failed" if attempt == MAX_RETRIES or result.error_output == last_error
            else "retry"
        )
        traces.append(AgentTrace(attempt=attempt, code=code_output.code,
                                  validation=result, status=status))

        if status == "success":
            log.info("All tests passed!")
            return traces

        if status == "failed":
            log.warning(f"Agent failed after {attempt} attempts")
            return traces

        # Convergence check
        last_error = result.error_output

        # Feed failure back to the model
        messages.append({
            "role": "user",
            "content": build_retry_prompt(attempt, result)
        })

    return traces
```

````python
# prompts.py

SYSTEM_PROMPT = """
You are an expert Python developer.
Write clean, correct Python code that passes the provided test suite.
Return a JSON object with:
  - "code": the complete Python implementation (no markdown fences)
  - "explanation": a brief explanation of your approach (1 paragraph)
"""

def build_user_prompt(task: str, test_suite: str) -> str:
    return f"""Task:
{task}

Test suite that your code must pass:
```python
{test_suite}
```

Write the implementation now."""

def build_retry_prompt(attempt: int, result) -> str:
    return f"""Your code failed the test suite (attempt {attempt}).

Test failures:
{result.error_output}

Fix the bugs and return corrected code. Return the full implementation, not just the changed parts."""
````

#### Reference solution

The two `NotImplementedError` stubs, completed:

```python
# validator.py
import subprocess
import tempfile
import os
from models import ValidationResult

def run_tests(code: str, test_suite: str, timeout: int = 30) -> ValidationResult:
    with tempfile.TemporaryDirectory() as tmp:
        with open(os.path.join(tmp, "solution.py"), "w") as f:
            f.write(code)
        with open(os.path.join(tmp, "tests.py"), "w") as f:
            f.write(test_suite)

        try:
            result = subprocess.run(
                ["pytest", "tests.py", "--tb=short", "-q"],
                cwd=tmp, capture_output=True, text=True, timeout=timeout,
            )
        except subprocess.TimeoutExpired:
            return ValidationResult(passed=False, error_output="Test run timed out",
                                    test_count=0, failed_count=0)

        output = result.stdout + result.stderr
        passed = result.returncode == 0

        # crude but reliable enough: pytest's summary line, e.g. "3 passed" / "1 failed, 2 passed"
        import re
        failed = sum(int(n) for n in re.findall(r"(\d+) failed", output))
        ok = sum(int(n) for n in re.findall(r"(\d+) passed", output))

        return ValidationResult(
            passed=passed,
            error_output=None if passed else output[-3000:],
            test_count=failed + ok,
            failed_count=failed,
        )
```

```python
# agent.py — generate_code
from openai import OpenAI
from pydantic import ValidationError

client = OpenAI()

def generate_code(messages: list) -> CodeOutput:
    response = client.chat.completions.create(
        model="gpt-4.1-mini",
        temperature=0,
        response_format={"type": "json_object"},
        messages=messages,
    )
    raw = response.choices[0].message.content
    try:
        return CodeOutput.model_validate_json(raw)
    except ValidationError as e:
        # Treat malformed output like a failed attempt: surface it as a structured error
        return CodeOutput(code="", explanation=f"Model returned invalid output: {e}")
```

#### Design notes

- **Why truncate `error_output` to the last 3,000 characters:** retry prompts must include the failure, but a huge traceback wastes context. The tail of pytest output contains the assertion details — the part the model needs.
- **Why `temperature=0`:** retries should converge. At higher temperatures the model can produce a different wrong answer each attempt, defeating the convergence check.
- **Why a `ValidationError` becomes an empty `CodeOutput` rather than an exception:** the hard constraint says no unhandled exceptions. Empty code fails the test run, which flows through the normal retry path with the validation error visible in the trace.
- **The convergence check is already in the starter `run_agent`:** `result.error_output == last_error` flips status to `failed`. The README should explain that this catches the "model is stuck in a loop producing the same bug" case that MAX_RETRIES alone would burn attempts on.

For the README's two stuck conditions: (1) the model repeats an identical wrong fix — bounded by the convergence check; (2) the task is unsolvable as specified (test suite contradicts the task) — bounded by MAX_RETRIES.
