# Module 06 - Multi-Agent Systems

**Part 3 — Orchestration & Multi-Agent** - 5 lessons + lab - ~5.5 hours total

<LearningObjectives>
- Explain the architectural difference between a parallel workflow and a true multi-agent system
- Identify the four agent roles — orchestrator, subagent, specialist, and critic — and describe what each one does
- Choose the right communication pattern for a given coordination requirement
- Apply zero-trust principles to inter-agent communication and explain why blind trust is dangerous
- Recognise the three categories of multi-agent failure from the MAST taxonomy and design mitigations for each
- Select between LangGraph, the Claude Agent SDK, AutoGen, and CrewAI based on your dominant constraint
- Design a four-agent research pipeline with defined roles, communication pattern, and failure mitigations
</LearningObjectives>

## What you'll build

You will design and implement a four-agent research pipeline in four different frameworks, then connect two independent multi-agent systems via a typed handoff schema. By the end you will have working code in LangGraph, the Claude Agent SDK, AutoGen, and CrewAI — and a cross-system pipeline showing agent-to-agent communication across a trust boundary.

<Callout type="key">
Multi-agent complexity is not free. Add it only when the task genuinely requires dynamic coordination — independent roles, parallel context windows, or mid-task replanning. For most tasks, a well-designed workflow is enough.
</Callout>

## Where this fits

Module 05 taught you how to compose steps into controlled workflows. This module extends that into systems where independent agents coordinate dynamically. Module 07 will cover evaluation — including how evaluation changes when multiple agents interact.
