## The mental model

AutoGen's core idea: agents communicate through a shared conversation thread. Instead of explicit tool calls or graph edges, agents take turns speaking — and the `GroupChatManager` decides who speaks next. This maps naturally to workflows that benefit from iterative refinement: one agent writes, another critiques, the first revises.

AG2 (AutoGen v0.4) introduced event-driven architecture and explicit graph workflows. This lab uses the GroupChat API (still the most widely used) and then shows the AG2 upgrade path.

## Step 1 — Define the agents

```python
import autogen

orchestrator = autogen.AssistantAgent(
    name="Orchestrator",
    system_message="""You are the research orchestrator. Your role:

1. DECOMPOSE: When given a research question, state the two sub-questions:
   - Sub-question A (for CompressionSpecialist): compression angle
   - Sub-question B (for InferenceSpecialist): inference angle
   Address each specialist by name when delegating.

2. COORDINATE: After both specialists have responded, say:
   "Critic, please review the outputs from CompressionSpecialist and InferenceSpecialist."

3. HANDLE VERDICT: If the critic says PASS, synthesise a final answer. 
   If FAIL, address the specific specialist with feedback and ask them to revise.
   Maximum 2 revision cycles.

4. SYNTHESISE: When ready, produce:
   FINAL_ANSWER:
   [your synthesis here]
   
   This terminates the conversation.
""",
    llm_config=llm_config,
    human_input_mode="NEVER",
)

compression_specialist = autogen.AssistantAgent(
    name="CompressionSpecialist",
    system_message="""You are a model compression research specialist.

SCOPE: Only quantisation (int8, fp16, GPTQ, AWQ), pruning, knowledge distillation, 
structured sparsity. Refuse any out-of-scope questions.

When delegated a research question by Orchestrator:
- Research your findings
- Structure your response:
  COMPRESSION FINDINGS:
  - Finding 1 [Citation: Author (Year)]
  - Finding 2 [Citation: Author (Year)]
  KEY CLAIM: [single most important finding]

If asked about inference, respond: "OUT_OF_SCOPE: This falls under inference optimisation. 
Please direct this to InferenceSpecialist."
""",
    llm_config=llm_config,
    human_input_mode="NEVER",
)

inference_specialist = autogen.AssistantAgent(
    name="InferenceSpecialist",
    system_message="""You are an LLM inference optimisation specialist.

SCOPE: Only speculative decoding, KV cache management, continuous batching, 
PagedAttention, hardware-aware serving. Refuse out-of-scope questions.

When delegated a research question by Orchestrator:
- Research your findings
- Structure your response:
  INFERENCE FINDINGS:
  - Finding 1 [Citation: Author (Year)]
  - Finding 2 [Citation: Author (Year)]
  KEY CLAIM: [single most important finding]

If asked about model compression, respond: "OUT_OF_SCOPE: This falls under compression. 
Please direct this to CompressionSpecialist."
""",
    llm_config=llm_config,
    human_input_mode="NEVER",
)

critic = autogen.AssistantAgent(
    name="Critic",
    system_message="""You are the research quality critic. You evaluate — you do not research.

When asked to review specialist outputs:
1. Check every factual claim has a citation
2. Check each specialist stayed in scope
3. Check for contradictions between the two outputs

Respond with:
VERDICT: PASS or FAIL
SCORE: 0.0-1.0
UNCITED_CLAIMS: [list any claims without citations]
ISSUES: [list any problems]
SUMMARY: [one sentence]

PASS threshold: score >= 0.75 AND no uncited claims.
""",
    llm_config=llm_config,
    human_input_mode="NEVER",
)

# UserProxy triggers the conversation and detects termination
user_proxy = autogen.UserProxyAgent(
    name="UserProxy",
    human_input_mode="NEVER",
    max_consecutive_auto_reply=20,
    is_termination_msg=lambda x: "FINAL_ANSWER:" in x.get("content", ""),
    code_execution_config=False,
)
```

## Step 2 — Configure the GroupChat

```python
groupchat = autogen.GroupChat(
    agents=[user_proxy, orchestrator, compression_specialist, inference_specialist, critic],
    messages=[],
    max_round=25,
    # "auto" lets the LLM decide who speaks next based on conversation context
    speaker_selection_method="auto",
    # Prevent the same agent from speaking twice in a row unnecessarily
    allow_repeat_speaker=False,
)

manager = autogen.GroupChatManager(
    groupchat=groupchat,
    llm_config=llm_config,
)
```

## Step 3 — Run the pipeline

```python
question = (
    "What is the current state of LLM efficiency research — "
    "specifically model compression and inference optimisation?"
)

chat_result = user_proxy.initiate_chat(
    manager,
    message=question,
    silent=False,  # Set True to suppress verbose output
)

# Extract the final answer
final_message = chat_result.chat_history[-1]["content"]
print(final_message)
```

## Step 4 — Inspect the conversation history

One of AutoGen's strengths is that the full conversation history is available for analysis.

```python
history = chat_result.chat_history

# Count turns per agent
from collections import Counter
turn_counts = Counter(msg["name"] for msg in history if "name" in msg)
print("Turns per agent:", dict(turn_counts))

# Find all critic verdicts
critic_turns = [msg for msg in history if msg.get("name") == "Critic"]
for turn in critic_turns:
    print("Critic verdict:", turn["content"][:200])

# Total token cost estimate
total_chars = sum(len(msg["content"]) for msg in history)
print(f"Approx tokens: ~{total_chars // 4:,}")
```

<Callout type="warning">
AutoGen's token cost compounds with conversation length. In a 4-agent, 20-round GroupChat, every agent call includes the full conversation history up to that point. A 10-round conversation where each round averages 500 tokens means round 10's context is ~5,000 tokens just from history — before any new content.
</Callout>

## Step 5 — The AG2 upgrade path

AG2 (AutoGen v0.4) introduces explicit graph-based workflows that are more predictable than speaker selection via LLM. Here is the same pipeline restructured as an AG2 graph:

```python
# AG2 graph-based approach (requires autogen-agentchat >= 0.4)
from autogen_agentchat.agents import AssistantAgent
from autogen_agentchat.teams import DiGraphGroupChat, DiGraphGroupChatManager
from autogen_agentchat.conditions import TextMentionTermination

# Define agents (same system prompts as above, different class)
orchestrator_a2 = AssistantAgent("Orchestrator", system_message="...", model_client=...)
compression_a2  = AssistantAgent("CompressionSpecialist", system_message="...", model_client=...)
inference_a2    = AssistantAgent("InferenceSpecialist", system_message="...", model_client=...)
critic_a2       = AssistantAgent("Critic", system_message="...", model_client=...)

# Define the directed graph explicitly
# Orchestrator → both specialists; specialists → critic; critic → orchestrator or END
team = DiGraphGroupChat(
    participants=[orchestrator_a2, compression_a2, inference_a2, critic_a2],
    edges={
        "Orchestrator":          ["CompressionSpecialist", "InferenceSpecialist"],
        "CompressionSpecialist": ["Critic"],
        "InferenceSpecialist":   ["Critic"],
        "Critic":                ["Orchestrator"],  # orchestrator handles verdict
    },
    termination_condition=TextMentionTermination("FINAL_ANSWER:"),
)

# Run — execution follows the graph, not LLM speaker selection
result = await team.run(task=question)
```

The AG2 graph approach trades the flexibility of LLM-selected speakers for the predictability of explicit edges — closer in spirit to LangGraph than to the original AutoGen GroupChat.

## Comparing the two approaches

| | GroupChat (AutoGen v0.2) | DiGraph (AG2 v0.4) |
|---|---|---|
| Speaker selection | LLM decides dynamically | Explicit graph edges |
| Predictability | Low — can go off-script | High — path is defined |
| Flexibility | High — handles unexpected task shapes | Medium — graph must cover all paths |
| Token cost | High — full history every call | Lower — only relevant context |
| Debugging | Hard — trace conversation log | Easier — edges define expected path |

<Solution>

#### Exercise 1 — token cost vs LangGraph

Use real usage numbers, not the character estimate — AutoGen exposes per-call usage via the client's cost tracking (`agent.client.total_usage_summary` on v0.2, or the `model_client` usage on v0.4); for LangGraph, sum `response.usage_metadata` over your node calls. Expected result: AutoGen lands **2–4× higher** for this 4-agent pipeline, and the gap *grows with rounds* — in GroupChat, round N's input includes all N−1 previous rounds, so cost is roughly quadratic in conversation length, while LangGraph nodes each see only the state fields you hand them (roughly linear). That asymmetry, not the absolute numbers, is the finding worth writing down.

#### Exercise 2 — round-robin failure

With `speaker_selection_method="round_robin"`, the turn order is fixed by the participants list, regardless of what the conversation needs. Concretely: after the orchestrator decomposes, the turn passes to the next agent in list order even if that's the inference specialist being asked the compression sub-question; the critic gets a turn *before both specialist outputs exist* and reviews half a pipeline; and when the critic fails an output, the "repair" turn goes to whoever is next in the rotation, not to the specialist that needs to fix its work. The pipeline doesn't crash — it produces confident nonsense on schedule, which is worse. Lesson: round-robin suits symmetric debates (every agent contributes the same kind of turn); any pipeline with *roles* needs either LLM selection (flexible, unpredictable) or explicit edges (AG2/LangGraph).

#### Exercise 3 — human agent

```python
user_proxy = UserProxyAgent(
    "Human",
    human_input_mode="ALWAYS",
    code_execution_config=False,
)
```

There's no fixed right answer; what you should observe: when you interject "Critic: be stricter — reject anything without a quantitative benchmark", the critic's *next* verdict applies your bar (your message is in the shared history every agent reads), and specialists adjust on their re-run. Two things worth noticing: your instruction persists in history, so the stricter bar applies to every later round whether you intended that or not, and each pause is a checkpoint where you can see exactly what every agent will see next — the GroupChat equivalent of `interrupt_before`.

#### Exercise 4 — AG2 graph with conditional repair routing

AG2 has no `add_conditional_edges`. Conditional routing is expressed by giving an edge an *activation condition* on the message content (in current AG2, `DiGraph` edges accept a condition; the idiom below) — which means **the critic must emit a routable token in its text**:

```python
# Critic system prompt gains:
# "If compression claims are uncited, include the token REPAIR_COMPRESSION.
#  If inference claims are uncited, include REPAIR_INFERENCE.
#  If both pass, include VERDICT_PASS."

from autogen_agentchat.teams import DiGraphBuilder, GraphFlow

builder = DiGraphBuilder()
builder.add_node(orchestrator_a2).add_node(compression_a2)
builder.add_node(inference_a2).add_node(critic_a2)

builder.add_edge(orchestrator_a2, compression_a2)
builder.add_edge(orchestrator_a2, inference_a2)
builder.add_edge(compression_a2, critic_a2)
builder.add_edge(inference_a2, critic_a2)

# Conditional edges: activated when the condition string appears in the message
builder.add_edge(critic_a2, compression_a2, condition="REPAIR_COMPRESSION")
builder.add_edge(critic_a2, inference_a2,   condition="REPAIR_INFERENCE")
builder.add_edge(critic_a2, orchestrator_a2, condition="VERDICT_PASS")

team = GraphFlow(
    participants=builder.get_participants(),
    graph=builder.build(),
    termination_condition=TextMentionTermination("FINAL_ANSWER:"),
)
```

The comparison the exercise asks for: in LangGraph, routing is a **Python function over typed state** — testable without a model, impossible to misspell silently. In AG2, routing is **string matching over model output** — the critic must be prompted to emit exact tokens, and a paraphrase ("compression needs repairs") silently activates nothing. If your AG2 graph stalls after the critic turn, that's almost always why. Cap repairs the same way as everywhere else: a max-rounds termination condition alongside the text-mention one.

</Solution>
