# Module 12 - Capstone

**Part 5 - Frontiers & Capstone** - 6 lessons + lab - ~8-10 hours total

<LearningObjectives>
- Select a realistic agent project with a clear user, scope, and success metric
- Produce an architecture plan that chooses agent, workflow, or multi-agent patterns deliberately
- Build an evaluation plan with golden cases, adversarial cases, and a launch threshold
- Harden the system with security, safety, cost, observability, and UX controls
- Prepare a deployed demo and launch-readiness review
- Present a capstone project with honest limitations and a concrete next iteration
</LearningObjectives>

## Why this module exists

The capstone is where the course stops being a sequence of concepts and becomes one coherent engineering project. You will choose a domain, design an agent system, build the minimum viable version, evaluate it, harden it, deploy it, and present the result.

This module is intentionally practical. A small, well-instrumented, well-evaluated agent beats an ambitious assistant that cannot be trusted.

## What you will ship

By the end of the capstone, you will have a working agent or agentic workflow, a deployed or locally runnable app, an architecture diagram, an evaluation report, a safety review, observability evidence, a cost estimate, a UX walkthrough, and a launch-readiness checklist.
