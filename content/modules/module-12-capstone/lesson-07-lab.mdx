# Lab - Final Capstone Submission

<LabBrief
  title="Final Capstone Submission"
  time="4-6 hours"
>

#### Objective

Ship a complete, reviewable agent project. Your submission should prove that the system works, that you know how to evaluate it, and that you understand what would be required before production launch.

#### Deliverables

- `README.md` - project overview, setup, and run instructions
- `architecture.md` - diagram and explanation of control flow, tools, permissions, and data
- `eval_report.md` - test set, rubric, results, and launch threshold
- `safety_review.md` - threat model, permission boundaries, approval gates, and residual risks
- `observability.md` - trace screenshot or sample logs, cost estimate, latency notes
- `demo.md` or demo video - walkthrough of happy path and one failure/edge case
- `launch_readiness.md` - final launch decision and next iteration plan

</LabBrief>

## Step 1 - Freeze scope

Write:

```md
This capstone ships:
- 
- 
-

This capstone does not ship:
- 
- 
-
```

Do not expand scope during the final lab. Fix quality before adding features.

## Step 2 - Run the evaluation set

Run your evaluation cases and report the number of cases, average score, pass/fail against launch threshold, top three failure modes, and changes made after evaluation.

## Step 3 - Run the hardening checklist

Mark each item Pass, Fail, or N/A. For every Fail, include impact, remediation, owner, and target date.

## Step 4 - Prepare the demo

Your demo must show input, agent run, output, trace or log evidence, one failure or edge case, and launch-readiness decision.

## Step 5 - Submit the launch decision

Use one of:

- ready for limited production
- ready for supervised pilot
- internal demo only
- not ready

Justify the decision with evidence from evaluation, safety review, and observability.
