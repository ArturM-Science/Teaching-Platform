# Lab · Multi-Tool Agent with Sandboxed Code Runner

<LabBrief title="Multi-Tool Agent" time="2–3 hours">

#### Objective

Build a working agent with three tools: web search, scoped file access, and a sandboxed code runner. Every tool must handle errors gracefully and have a JSON schema that rejects at least one class of invalid input.

#### Setup

```bash
pip install openai httpx jsonschema
export OPENAI_API_KEY="sk-..."
```

Create a workspace directory the file tool will be scoped to:

```bash
mkdir -p ./workspace
```

#### Deliverables

- `tools.py` — all three tools with schemas and validation
- `agent.py` — the calling loop with MAX_STEPS and error handling
- `test_tools.py` — at least one test per tool showing it rejects an invalid input
- One paragraph in a `README.md` explaining the blast radius of each tool and how you bounded it

#### Hard constraints

- The file tool must reject any path that resolves outside `./workspace/`
- The code runner must time out after 10 seconds maximum
- No tool may raise an unhandled Python exception — all errors must return a structured error dict
- MAX_STEPS must be set and must be no more than 15

</LabBrief>

```python
# agent.py
import json
import openai
from tools import TOOLS, TOOL_MAP

client = openai.OpenAI()
MAX_STEPS = 10

def run_agent(user_message: str) -> str:
    messages = [{"role": "user", "content": user_message}]

    for step in range(MAX_STEPS):
        response = client.chat.completions.create(
            model="gpt-4o",
            messages=messages,
            tools=TOOLS,
        )
        msg = response.choices[0].message
        messages.append(msg)

        if not msg.tool_calls:
            return msg.content

        for call in msg.tool_calls:
            args = json.loads(call.function.arguments)
            # TODO: add argument validation before executing
            result = TOOL_MAP[call.function.name](**args)
            messages.append({
                "role": "tool",
                "tool_call_id": call.id,
                "content": json.dumps(result)
            })

    return "Agent reached maximum steps without completing."

if __name__ == "__main__":
```

The three tool implementations:

```python
def search_web(query: str, max_results: int = 5) -> dict:
    try:
        resp = httpx.get(
            "https://api.duckduckgo.com/",
            params={"q": query, "format": "json", "no_html": 1},
            timeout=10,
        )
        resp.raise_for_status()
        data = resp.json()
        results = [
            {"title": t.get("Text", ""), "url": t.get("FirstURL", "")}
            for t in data.get("RelatedTopics", [])[:max_results]
            if isinstance(t, dict) and t.get("FirstURL")
        ]
        return {"results": results}
    except httpx.TimeoutException:
        return {"error": "TimeoutError", "message": "Search timed out", "retryable": True}
    except httpx.HTTPStatusError as e:
        return {"error": "HTTPError", "message": str(e),
                "retryable": e.response.status_code >= 500}

def read_file(path: str) -> dict:
    resolved = os.path.realpath(os.path.join(WORKSPACE, path))
    if not resolved.startswith(WORKSPACE + os.sep) and resolved != WORKSPACE:
        return {"error": "PathError", "message": "Path outside workspace", "retryable": False}
    try:
        with open(resolved, "r", encoding="utf-8") as f:
            return {"content": f.read(20_000)}
    except FileNotFoundError:
        return {"error": "NotFound", "message": f"No such file: {path}", "retryable": False}
    except OSError as e:
        return {"error": "OSError", "message": str(e), "retryable": False}

def run_code(code: str, timeout: int = 10) -> dict:
    timeout = min(timeout, 10)
    try:
        result = subprocess.run(
            ["python", "-c", code],
            cwd=WORKSPACE, capture_output=True, text=True, timeout=timeout,
        )
        return {"stdout": result.stdout[-5000:], "stderr": result.stderr[-5000:],
                "returncode": result.returncode}
    except subprocess.TimeoutExpired:
        return {"error": "TimeoutError",
                "message": f"Script exceeded {timeout}s", "retryable": False}
```

Argument validation in the agent loop (the `# TODO: add argument validation` line):

```python
        for call in msg.tool_calls:
            name = call.function.name
            try:
                args = json.loads(call.function.arguments)
            except json.JSONDecodeError:
                result = {"error": "BadArguments", "message": "Arguments were not valid JSON"}
            else:
                if name not in TOOL_MAP:
                    result = {"error": "UnknownTool", "message": f"No such tool: {name}"}
                else:
                    schema = next(t for t in TOOLS if t["function"]["name"] == name)
                    try:
                        jsonschema.validate(args, schema["function"]["parameters"])
                        result = TOOL_MAP[name](**args)
                    except jsonschema.ValidationError as e:
                        result = {"error": "SchemaError", "message": e.message}

            messages.append({
                "role": "tool",
                "tool_call_id": call.id,
                "content": json.dumps(result)
            })
```

#### Why this passes the constraints

- **Path traversal:** `os.path.realpath` resolves `..` sequences and symlinks *before* the prefix check, so `../../etc/passwd` resolves outside `WORKSPACE` and is rejected. The string-only check the rubric warns about would miss symlinks.
- **Timeout:** `min(timeout, 10)` enforces the cap even if the model requests more; `TimeoutExpired` becomes a structured error, marked `retryable: False` because retrying an infinite loop wastes 10 more seconds.
- **No unhandled exceptions:** every failure path — bad JSON args, unknown tool, schema violation, tool error — flows back to the model as a `tool` message, so the agent can recover or report instead of crashing.
- **Output truncation:** `stdout`/`stderr`/file content are truncated so one chatty script can't flood the context window.
