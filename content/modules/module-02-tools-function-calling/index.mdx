# Module 2 · Tools & Function Calling

**Part 2 — A Single Capable Agent** · 5 lessons · ~5 hours total

<LearningObjectives>
- Design well-scoped tools with clear JSON schemas and descriptions that guide the model correctly
- Trace the full multi-turn function-calling protocol — from schema definition through result handling
- Connect an agent to an MCP server and understand when the protocol is worth adopting
- Handle tool failures gracefully with proper error objects, retry caps, and exponential backoff
- Apply the principle of least privilege and blast-radius thinking before giving an agent any real capability
</LearningObjectives>

## What you'll build

A working agent that can search the web, read and write files in a scoped directory, and execute code in a sandboxed subprocess — with proper error handling on every tool and JSON schemas that reject invalid inputs before the model ever sees a bad result. No frameworks. Plain Python and the OpenAI function-calling API.

This is the module where agents gain real-world capability. That means security is live from lesson 1.
