# Module 05 - Workflow Patterns & Control Flow

**Part 3 — Orchestration & Multi-Agent** - 5 lessons + lab - ~5.5 hours total

<LearningObjectives>
- Explain when a workflow is the right unit of composition instead of a single prompt call
- Build sequential chains that compress context between steps to avoid window overflow
- Implement classifier-based routing that sends requests to the right specialist
- Write verification loops with explicit stopping conditions that cannot run forever
- Fan work out to parallel branches and aggregate results safely
- Deliver a working multi-step research workflow with documented control flow and bounded repair
</LearningObjectives>

## What you'll build

You will build a three-stage research workflow: a planner that turns a question into structured queries, a parallel searcher that fans out and fans back in, and a synthesiser with a bounded verification loop that catches missing citations. The final lab asks you to wire all stages together, document the control flow, and identify one failure mode your stopping condition protects against.

<Callout type="key">
A workflow is not a better prompt. It is a different unit of composition — one that adds structure, verification, and control to work that a single call cannot reliably complete.
</Callout>

## Where this fits

Module 04 taught you how to evaluate a single agent. This module teaches you how to compose multiple steps into a controlled pipeline. Module 06 extends that into full multi-agent systems where independent agents coordinate.
