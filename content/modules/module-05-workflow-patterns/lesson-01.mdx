# Lesson 1 - From Prompt to Workflow

A single prompt call is the right tool for most tasks. When it isn't, the failure is usually visible: the output is too shallow, the context window fills before the work is done, or an error in step two propagates silently into the final answer. Workflows exist to fix those specific problems — not to add structure for its own sake.

## When a single call breaks down

A prompt call has one context window, one output, and no checkpoints. That works well when the task is self-contained. It breaks when:

- **The task is too long for one context window.** A single call cannot process a 200-page document and write a synthesis; it has to drop content or hallucinate what it did not see.
- **Different steps need different capabilities.** Classifying a support ticket and then drafting a reply are different problems. A single prompt trying to do both degrades at both.
- **Errors compound silently.** If step one produces a wrong assumption, step two builds on it. There is no checkpoint between them to catch the drift.
- **You need to verify before continuing.** A research step should be checked before feeding its output into a legal summary. A single call has no internal gate.

## What a workflow adds

A workflow is a directed set of steps where the output of one step becomes the input of the next, with control logic deciding which step runs when.

```
input → [step A] → [step B] → [step C] → output
                       ↑
                   [branch condition]
```

Three things a workflow adds that a single call cannot:

| What | Why it matters |
|---|---|
| **Checkpoints** | Verify output before passing it forward |
| **Branching** | Route to different steps based on what happened |
| **Bounded loops** | Retry or repair until a condition is met, then stop |

## The composition spectrum

Not every multi-step task needs a full workflow engine. The options from simplest to most complex:

| Option | When to use |
|---|---|
| Single call | Self-contained task, fits in one context window |
| Prompt chain | 2-4 sequential steps, no branching needed |
| Stateful workflow | Branching, loops, or verification required |
| Multi-agent system | Independent agents with separate roles and context |

Start at the simplest option that meets the requirement. A workflow you do not need is overhead you will maintain forever.

## When NOT to build a workflow

A workflow adds coordination cost. Before adding one, ask:

- Can a longer or better-structured prompt handle this?
- Is the multi-step nature from the problem itself, or from an underspecified prompt?
- Will the extra steps add latency that matters to the user?

The most common mistake is building a five-step pipeline for a task that a single well-crafted prompt would handle in one call.
