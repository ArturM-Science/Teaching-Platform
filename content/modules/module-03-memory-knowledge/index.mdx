# Module 3 · Memory & Knowledge

**Part 2 — A Single Capable Agent** · 5 lessons · ~5 hours total

<LearningObjectives>
- Explain why the context window is not memory and choose the right memory type for a given piece of information
- Build a vector database with ChromaDB, embed documents, and retrieve by semantic similarity
- Implement a complete RAG pipeline from chunking through grounded generation
- Diagnose retrieval failures and implement agentic RAG with query reformulation and reflection
- Build a long-term memory system using an Obsidian vault that agents read, write, and consolidate across sessions
</LearningObjectives>

## What you'll build

A research agent with persistent long-term memory backed by an Obsidian vault. The agent searches its memory before hitting the web, writes new knowledge as linked markdown notes, indexes the vault with ChromaDB for semantic retrieval, and runs a consolidation pass that merges near-duplicate notes and archives stale ones. After multiple sessions, the vault grows as a navigable, human-readable record of everything the agent has learned.
