# Module 13 - OpenAI Codex: Agentic Software Engineering

**Part 6 - Practitioner Tools** - 6 lessons - ~5 hours total

<LearningObjectives>
- Explain Codex as a coding agent: repo context, tool use, editing, shell commands, verification, and human review
- Write task prompts that include scope, context, constraints, and a concrete definition of done
- Choose safe permissions for local and cloud coding-agent work
- Use `AGENTS.md`, skills, and MCP to make a coding agent follow project-specific rules
- Apply subagents, code review, and repair loops without creating conflicting edits or noisy context
- Complete a real repo change with Codex while preserving a reviewable Git trail
</LearningObjectives>

## What you'll build

You will use Codex as a supervised software engineering agent on a real repository. The final lab is not "ask the model to write code and hope." You will create a branch, ask Codex to understand the codebase, write a precise task spec, let it make a small change, run verification, ask for a review, fix one issue, and capture the durable project rule in `AGENTS.md`.

The point of this module is not that Codex is magic. The point is that a coding agent becomes useful when the engineer knows how to bound it, steer it, and verify it.
