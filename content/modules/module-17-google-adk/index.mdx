# Google Agent Development Kit (ADK)

**Framework Elective · Optional · Prerequisite: Module 06**

Google's Agent Development Kit (ADK 2.0) is built around one central idea: agents form a tree, and the root delegates to branches. That hierarchical model — combined with one-command deployment to Vertex AI and the A2A (Agent-to-Agent) open protocol for cross-framework calls — makes ADK the right choice when you need auditability, managed infrastructure, and the ability to call LangGraph or CrewAI agents as peers.

<Callout type="note">
This module assumes you are comfortable with Module 06 (Multi-Agent Systems). You should know what an orchestrator is, what fan-out/fan-in means, and why session state matters in multi-agent pipelines. If those terms are unfamiliar, complete Module 06 first.
</Callout>

---

## What You Will Build

By the end of this module you will have a **Multi-Agent Research System**: a three-tier ADK pipeline that:

1. Decomposes a research question with a planner (`LlmAgent`)
2. Searches web and academic sources concurrently (`ParallelAgent`)
3. Synthesises findings into a draft and runs a critic quality gate (`LoopAgent`)
4. Persists session state across restarts with `DatabaseSessionService` (SQLite)
5. Is verifiable locally with `adk dev` and deployable to Vertex AI with `adk deploy`

## Setup

Python 3.10–3.13 required. All code examples are tested against ADK 2.2.x.

**Step 1 — Create and activate a virtual environment**

```bash
# Create the venv (do this once per project)
python -m venv .venv

# Activate it — macOS / Linux
source .venv/bin/activate

# Activate it — Windows (PowerShell)
.venv\Scripts\Activate.ps1

# Activate it — Windows (Command Prompt)
.venv\Scripts\activate.bat
```

You should see `(.venv)` in your prompt. All packages installed after this point stay inside the venv and won't affect your system Python.

**Step 2 — Install dependencies**

```bash
pip install "google-adk>=2.2.0"

# For Vertex AI deployment (Lesson 5 and Lab — optional):
pip install "google-cloud-aiplatform[adk]"

# Authenticate with Google (for Vertex AI only):
gcloud auth application-default login
```

**Step 3 — Store your API key in a `.env` file**

Get a free key at [aistudio.google.com](https://aistudio.google.com).

First, add `.env` to `.gitignore` so the key is never committed:

```bash
echo ".env" >> .gitignore
```

Then create a `.env` file at the root of your project with this content:

```
GOOGLE_API_KEY=your-google-ai-studio-key
```

Install `python-dotenv` so your scripts can load it:

```bash
pip install python-dotenv
```

At the top of every Python script or notebook, add these two lines **before** any ADK imports:

```python
from dotenv import load_dotenv
load_dotenv()
```

`load_dotenv()` reads `.env` and injects the values into `os.environ`. Any code that calls `os.getenv("GOOGLE_API_KEY")` after this point will find the key automatically — including the ADK internals.

<Callout type="warning">
Always add `.env` to `.gitignore` before creating the file. If you commit it first, rotate your key immediately at [aistudio.google.com](https://aistudio.google.com) — keys in git history are compromised even after deletion.
</Callout>
