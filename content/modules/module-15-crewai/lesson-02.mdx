## A complete sequential crew (60 lines)

This crew answers technical questions with a three-stage pipeline:

1. **Researcher** Гўв‚¬вЂќ finds and synthesises information
2. **Analyst** Гўв‚¬вЂќ evaluates what the researcher found
3. **Writer** Гўв‚¬вЂќ produces the final report

```python
from crewai import Agent, Task, Crew, Process
from langchain_anthropic import ChatAnthropic

# ГўвЂќв‚¬ГўвЂќв‚¬ LLM ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬
llm = ChatAnthropic(model="claude-haiku-4-5-20251001")

# ГўвЂќв‚¬ГўвЂќв‚¬ Agents ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬
researcher = Agent(
    role="Senior Technical Researcher",
    goal="Find precise, well-sourced answers to technical questions",
    backstory=(
        "You have a decade of experience reviewing academic papers and "
        "technical reports. You cite specific numbers and flag uncertainty."
    ),
    llm=llm,
    allow_delegation=False,
    verbose=True,
    max_iter=5,
)

analyst = Agent(
    role="Critical Analyst",
    goal="Identify gaps, risks, and practical implications in research",
    backstory=(
        "You specialize in translating raw research into actionable insights. "
        "You are skeptical by nature and always ask 'what could go wrong?'"
    ),
    llm=llm,
    allow_delegation=False,
    verbose=True,
    max_iter=5,
)

writer = Agent(
    role="Technical Report Writer",
    goal="Produce clear, structured reports that non-specialists can act on",
    backstory=(
        "You've written hundreds of technical briefings for executive audiences. "
        "You know how to trim jargon without losing precision."
    ),
    llm=llm,
    allow_delegation=False,
    verbose=True,
    max_iter=3,
)

# ГўвЂќв‚¬ГўвЂќв‚¬ Tasks ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬
research_task = Task(
    description=(
        "Research the current state of {topic}. "
        "Identify the 5 most important recent developments."
    ),
    expected_output=(
        "A numbered list of 5 developments. Each entry must include: "
        "the development name, a 2-sentence explanation, "
        "and a specific data point or citation."
    ),
    agent=researcher,
)

analysis_task = Task(
    description="Analyse the research findings and identify the top 3 practical risks.",
    expected_output=(
        "A structured analysis with three numbered risks. "
        "Each risk must have: a title, a severity rating (High/Medium/Low), "
        "and a 2-sentence explanation of the practical implication."
    ),
    agent=analyst,
    context=[research_task],
)

report_task = Task(
    description=(
        "Write a concise executive report on {topic} "
        "based on the research and risk analysis."
    ),
    expected_output=(
        "A 4-section report: (1) Summary (3 sentences), "
        "(2) Key Findings (bullet list), "
        "(3) Top Risks (from the analysis), "
        "(4) Recommended Next Steps (3 bullets)."
    ),
    agent=writer,
    context=[research_task, analysis_task],
    output_file="report.md",
)

# ГўвЂќв‚¬ГўвЂќв‚¬ Crew ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬
crew = Crew(
    agents=[researcher, analyst, writer],
    tasks=[research_task, analysis_task, report_task],
    process=Process.sequential,
    verbose=True,
)

# ГўвЂќв‚¬ГўвЂќв‚¬ Run ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬ГўвЂќв‚¬
result = crew.kickoff(inputs={"topic": "large language model efficiency"})

print("\n=== Final Report ===")
print(result.raw)

print("\n=== Token Usage ===")
print(crew.usage_metrics)
```

Run it:

```bash
python crew.py
```

You'll see verbose output for each agent as it works through its task Гўв‚¬вЂќ what the agent is thinking, what tools it uses, and what output it produces before moving on.

## Key configuration options explained

### allow_delegation

```python
agent = Agent(..., allow_delegation=False)
```

When `allow_delegation=True`, an agent can ask other agents for help Гўв‚¬вЂќ even in sequential mode. This is most meaningful in hierarchical mode, where the manager controls delegation. In sequential mode with no manager, delegation rarely does what you expect. Set it to `False` for most agents in `Process.sequential`.

**Rule of thumb:** `allow_delegation=False` unless you're using `Process.hierarchical` and you specifically want worker agents to be able to consult each other.

### verbose

```python
agent = Agent(..., verbose=True)
crew = Crew(..., verbose=True)
```

`verbose=True` on an agent prints the agent's internal reasoning and tool calls. `verbose=True` on the crew prints higher-level orchestration events. Use both in development so you can see exactly what's happening. Set `verbose=False` in production Гўв‚¬вЂќ the stdout overhead adds latency.

### max_iter

```python
agent = Agent(..., max_iter=5)
```

`max_iter` limits how many tool-call iterations an agent can make within a single task. The default is 5. If an agent has tools and tends to loop Гўв‚¬вЂќ calling the same tool repeatedly Гўв‚¬вЂќ lower this value. If your tasks are complex and require many tool calls, raise it. Without tools, this setting has no effect.

<Callout type="tip">
Run your first crew with `verbose=True` on both agents and the crew so you can see exactly what each agent is doing and which tools it calls. Once it works reliably, set `verbose=False` for production. The verbose output is one of CrewAI's best debugging tools Гўв‚¬вЂќ use it early.
</Callout>

## Variable interpolation with kickoff(inputs=...)

Any `{placeholder}` in a task's `description` or `expected_output` is replaced at runtime with the matching key from `inputs`:

```python
research_task = Task(
    description="Research the current state of {topic}.",
    expected_output="A list of 5 key developments in {topic}.",
    agent=researcher,
)

result = crew.kickoff(inputs={"topic": "quantum computing"})
```

At runtime, both strings become:

```
"Research the current state of quantum computing."
"A list of 5 key developments in quantum computing."
```

This makes your crews reusable across topics or domains without changing any code. You can use multiple placeholders:

```python
Task(description="Write a {length}-word summary of {topic} for a {audience} audience.")
```

Then: `crew.kickoff(inputs={"length": "200", "topic": "AI safety", "audience": "policy maker"})`

## Reading output: result.raw vs result.pydantic

`crew.kickoff()` returns a `CrewOutput` object.

**`result.raw`** Гўв‚¬вЂќ always available. The raw string output of the last task in the crew.

```python
result = crew.kickoff(inputs={"topic": "..."})
print(result.raw)   # "## Executive Report\n\n### Summary\n..."
```

**`result.pydantic`** Гўв‚¬вЂќ available only when the last task has `output_pydantic` set to a Pydantic model. The output is parsed and validated into a model instance.

```python
from pydantic import BaseModel
from typing import List

class Report(BaseModel):
    summary: str
    key_findings: List[str]
    risks: List[str]
    next_steps: List[str]

report_task = Task(
    description="Write an executive report...",
    expected_output="JSON with keys: summary, key_findings, risks, next_steps.",
    agent=writer,
    output_pydantic=Report,
)

result = crew.kickoff(inputs={"topic": "..."})
print(result.pydantic.summary)          # typed string
print(result.pydantic.key_findings)     # typed list
```

If the LLM output can't be parsed into the model, `result.pydantic` is `None` and `result.raw` still contains the raw string.

## Saving outputs to files

Any task can write its output to a file automatically:

```python
report_task = Task(
    description="Write a report...",
    expected_output="...",
    agent=writer,
    output_file="outputs/report.md",
)
```

CrewAI writes the task's final output to the specified path after the task completes. The file is created relative to the working directory. This is useful for long-running crews where you want outputs persisted even if the script crashes before the final print.

## Inspecting token usage

```python
result = crew.kickoff(inputs={"topic": "..."})
print(crew.usage_metrics)
```

Output:

```
UsageMetrics(
    total_tokens=4821,
    prompt_tokens=3944,
    completion_tokens=877,
    successful_requests=3
)
```

`total_tokens` is your cost driver. `successful_requests` matches the number of LLM calls made Гўв‚¬вЂќ useful for checking whether agents made unexpected extra calls (which can indicate tool-use loops).

You can compare metrics across runs to understand how different configurations affect cost:

```python
# Run 1: tight expected_output descriptions
crew_a.kickoff(inputs=inputs)
print("Tight descriptions:", crew_a.usage_metrics.total_tokens)

# Run 2: vague expected_output descriptions
crew_b.kickoff(inputs=inputs)
print("Vague descriptions:", crew_b.usage_metrics.total_tokens)
```

Tight `expected_output` descriptions typically reduce tokens because agents spend fewer iterations reformatting their output.

<Callout type="note">
`crew.usage_metrics` reflects all LLM calls made by the crew Гўв‚¬вЂќ including any retries, tool-use steps, and manager calls in hierarchical mode. It is reset each time you call `crew.kickoff()`, so if you run the same crew twice in a script, `usage_metrics` reflects only the most recent run.
</Callout>

## Putting it all together: what to verify

After running the crew, check these things:

1. **Did each agent complete its task?** Look for "Task completed" in verbose output.
2. **Does `result.raw` match the format described in `expected_output`?** If not, tighten the description.
3. **Are `successful_requests` what you expected?** 3 agents ГѓвЂ” 1 task each = at least 3 requests. More requests mean tool calls or retries happened.
4. **Is `output_file` created?** Check the working directory.
