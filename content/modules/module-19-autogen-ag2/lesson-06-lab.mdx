# Lab: Iterative Code Review System with Nested Chats

In Lessons 2 and 3 you learned how to build GroupChats and wire nested chats onto them. In this lab you build a realistic quality-control pipeline that uses nested chats — not GroupChat — as its primary coordination mechanism.

The system has four agents. A **Developer** writes code in response to a specification. After every Developer message, a **Reviewer** fires in a nested chat to check code quality. If the Reviewer approves, a **Security Auditor** fires in a second nested chat to check for vulnerabilities. The **Orchestrator** manages the outer loop and applies the `APPROVED:` termination condition.

This architecture models a real engineering team where every piece of code goes through code review and security review before it is accepted.

## Hard constraints

Before you start, read these — they are graded:

- Must use nested chats (not GroupChat) for the review pipeline
- Termination condition: `"APPROVED: "` followed by the agent name (e.g., `"APPROVED: Developer"`)
- `human_input_mode="NEVER"` for all agents — fully automated
- `max_consecutive_auto_reply=6` for the Orchestrator
- Chat history must be saved to `review_trace.json` after each run
- No hardcoded API keys — read from environment variables only

```python
# nested_chats.py
from agents import orchestrator, developer, reviewer, security_auditor


def build_review_message(recipient, messages, sender, config):
    last_code = messages[-1]["content"]
    return (
        f"Please review this Python code for quality, readability, and edge case handling:\n\n"
        f"{last_code}"
    )


def build_security_message(recipient, messages, sender, config):
    # Walk back to find the Developer's last message (code)
    for msg in reversed(messages):
        if msg.get("name") == "Developer":
            return (
                f"This code has passed quality review. Please audit it for security vulnerabilities:\n\n"
                f"{msg['content']}"
            )
    return f"Please audit this code for security vulnerabilities:\n\n{messages[-1]['content']}"


def should_run_security_audit(messages) -> bool:
    if not messages:
        return False
    last = messages[-1].get("content", "")
    return "Code quality: APPROVED" in last


def register_pipeline(orchestrator, developer, reviewer, security_auditor):
    orchestrator.register_nested_chats(
        chat_queue=[
            {
                "recipient": reviewer,
                "message": build_review_message,
                "max_turns": 1,
                "summary_method": "last_msg",
            },
            {
                "recipient": security_auditor,
                "message": build_security_message,
                "max_turns": 1,
                "summary_method": "last_msg",
                "condition": should_run_security_audit,  # only runs if reviewer approved
            },
        ],
        trigger=developer,
    )
```

```python
# runner.py
import json
import datetime
from agents import orchestrator, developer, reviewer, security_auditor
from nested_chats import register_pipeline

SPECIFICATION = """
Write a Python function called `safe_divide(a, b)` that:
1. Takes two numbers as input
2. Returns the result of a / b
3. Raises a ValueError with a helpful message if b is zero
4. Includes a docstring
5. Has a simple __main__ block that demonstrates three cases:
   - Normal division
   - Division that produces a float
   - Division by zero (caught and printed)
"""


def save_trace(orchestrator, recipient):
    messages = orchestrator.chat_messages.get(recipient, [])
    path = "review_trace.json"
    with open(path, "w") as f:
        json.dump(messages, f, indent=2)
    print(f"Trace saved to {path} ({len(messages)} messages)")


def main():
    register_pipeline(orchestrator, developer, reviewer, security_auditor)

    try:
        orchestrator.initiate_chat(
            developer,
            message=f"Implement the following specification:\n\n{SPECIFICATION}",
        )
    except Exception as e:
        print(f"Chat failed: {type(e).__name__}: {e}")
    finally:
        save_trace(orchestrator, developer)

    # Find the approved message
    messages = orchestrator.chat_messages.get(developer, [])
    approved = next(
        (m["content"] for m in reversed(messages) if "APPROVED: Developer" in m.get("content", "")),
        None,
    )
    if approved:
        print("\n=== APPROVED CODE ===")
        print(approved)
    else:
        print("\nNo approved message found — check review_trace.json for the full history.")


if __name__ == "__main__":
    main()
```

## Step 5: Extension challenges

Once your base system is working, try these:

**Extension A — Add a third review stage.** Add a **DocumentationAgent** that fires after the Security Auditor approves and checks that the docstrings are complete and accurate. The full pipeline becomes: Reviewer → SecurityAuditor → DocumentationAgent.

**Extension B — Use round_robin for the outer loop.** Swap the outer `initiate_chat` conversation between Orchestrator and Developer to a GroupChat with `round_robin` selection. Measure whether this changes the total number of LLM calls.

**Extension C — Cost instrumentation.** After each run, parse `review_trace.json` and count:
- Total messages in the outer conversation
- Total nested chat messages (you will need to log them separately)
- Estimated token count based on message length

Print a cost summary at the end of the run.
