# Lab — Resumable Research Pipeline with LangGraph

<LabBrief title="Resumable Research Pipeline with LangGraph" time="2–3 hours">
#### Objective
Build a 5-node research pipeline with full `SqliteSaver` checkpointing, parallel researchers via the Send API, a critic that can trigger repairs, and a human-in-the-loop approval gate before synthesis. The pipeline must survive a process restart and resume from checkpoint.

#### Setup
```bash
pip install langgraph langchain-anthropic "langgraph[sqlite]"
export ANTHROPIC_API_KEY="sk-ant-..."
```

#### Deliverables
- `state.py` — `ResearchState` TypedDict with reducer for parallel findings
- `nodes.py` — all 5 node functions: planner, researcher, critic, synthesiser, plus dispatch routing
- `pipeline.py` — full LangGraph graph with SqliteSaver checkpointing
- `runner.py` — CLI that streams node updates, pauses for human approval, and resumes
- `README.md` — ASCII/Mermaid graph diagram and one paragraph explaining what `interrupt_before` gives you that a simple `input()` check inside a node does not

#### Hard constraints
- Must use `SqliteSaver` — persistence must survive a process restart
- Parallel researchers must use the Send API (not static fan-out edges)
- Must use `interrupt_before=["synthesiser"]` for human approval
- Runner must print state updates after each node (`stream_mode="updates"`)
- Retry guard: max 2 critic-triggered repairs (tracked via `repair_count` in state)
- No hardcoded API keys — all secrets via environment variables
</LabBrief>

## Pipeline Architecture

```
                    ┌─────────────────────────────┐
                    │         PLANNER              │
                    │  generates N search queries  │
                    └────────────┬────────────────┘
                                 │  Send API (dynamic fan-out)
              ┌──────────────────┼──────────────────┐
              ▼                  ▼                  ▼
         RESEARCHER          RESEARCHER          RESEARCHER
         (query 1)           (query 2)           (query N)
              │                  │                  │
              └──────────────────┴──────────────────┘
                          findings merged by reducer
                                 │
                    ┌────────────▼────────────────┐
                    │          CRITIC              │
                    │   evaluates all findings     │
                    └────────────┬────────────────┘
                                 │
                    ┌────────────▼────────────────┐
               ┌── no, repair_count < 2           │
               │    route back to PLANNER         │
               │                                  │
               └──────────────────────────────────┘
                    ┌────────────▼────────────────┐
                    │   [interrupt_before here]   │
                    │       SYNTHESISER            │
                    │   human approves before      │
                    │    this node runs            │
                    └────────────┬────────────────┘
                                 │
                               END
```

**Five nodes**: `planner` → (Send API) → `researcher` → `critic` → [human gate] → `synthesiser`

<details>
<summary>Solution: state.py — reducer annotation</summary>

```python
from typing import Annotated, TypedDict
import operator


class ResearchState(TypedDict):
    topic: str
    queries: list[str]
    findings: Annotated[list[str], operator.add]   # reducer: append from parallel workers
    critique: str
    critique_passed: bool
    repair_count: int
    human_feedback: str
    report: str


class WorkerState(TypedDict):
    query: str
    findings: Annotated[list[str], operator.add]
```

The key insight: `Annotated[list[str], operator.add]` tells LangGraph to concatenate lists when two parallel branches both return `{"findings": [...]}`. Without this, the last branch's findings overwrite all others.
</details>

<details>
<summary>Solution: nodes.py — planner</summary>

```python
def planner(state: ResearchState) -> dict:
    prompt = (
        f"Generate exactly 3 distinct research sub-questions about:\n{state['topic']}\n\n"
        f"Return only the questions, one per line, no numbering, no preamble."
    )
    response = model.invoke(prompt)
    queries = [q.strip() for q in response.content.strip().split("\n") if q.strip()][:3]
    return {
        "queries": queries,
        "findings": [],          # reset findings for this attempt
        "repair_count": state.get("repair_count", 0),
    }
```
</details>

<details>
<summary>Solution: nodes.py — researcher</summary>

```python
def researcher(state: WorkerState) -> dict:
    response = model.invoke(
        f"Research this question thoroughly in 3-4 sentences:\n{state['query']}"
    )
    return {"findings": [response.content]}
```

Returning a single-item list `[response.content]` is intentional. The reducer (`operator.add`) concatenates lists from all parallel workers. If all three workers return `["finding"]`, the final state has `findings = ["finding1", "finding2", "finding3"]`.
</details>

def critic(state: ResearchState) -> dict:
    findings_text = "\n".join(f"- {f}" for f in state["findings"])
    prompt = (
        f"Evaluate whether these findings sufficiently answer the topic.\n\n"
        f"Topic: {state['topic']}\n\nFindings:\n{findings_text}\n\n"
        f"Reply with PASS or FAIL followed by a one-sentence explanation."
    )
    text = model.invoke(prompt).content
    return {
        "critique": text,
        "critique_passed": text.strip().upper().startswith("PASS"),
        "repair_count": state["repair_count"] + 1,
    }

<summary>Solution: pipeline.py — build_graph</summary>

```python
def build_graph(checkpointer):
    builder = StateGraph(ResearchState)

    builder.add_node("planner", planner)
    builder.add_node("researcher", researcher)
    builder.add_node("critic", critic)
    builder.add_node("synthesiser", synthesiser)

    builder.set_entry_point("planner")

    builder.add_conditional_edges("planner", dispatch_researchers, ["researcher"])
    builder.add_edge("researcher", "critic")
    builder.add_conditional_edges(
        "critic",
        route_after_critic,
        {"synthesiser": "synthesiser", "planner": "planner"},
    )
    builder.add_edge("synthesiser", END)

    return builder.compile(
        checkpointer=checkpointer,
        interrupt_before=["synthesiser"],
    )
```

<summary>Solution: runner.py — streaming and resume</summary>

```python
def run_pipeline(topic: str, thread_id: str | None = None):
    graph, checkpointer = get_graph()

    if thread_id is None:
        thread_id = str(uuid.uuid4())
        print(f"Starting new run: thread_id={thread_id}")
    else:
        print(f"Resuming run: thread_id={thread_id}")

    config = {"configurable": {"thread_id": thread_id}}

    initial_state: ResearchState = {
        "topic": topic, "queries": [], "findings": [],
        "critique": "", "critique_passed": False,
        "repair_count": 0, "human_feedback": "", "report": "",
    }

    print("\n--- Phase 1: Research ---")
    for update in graph.stream(initial_state, config=config, stream_mode="updates"):
        node_name = list(update.keys())[0]
        node_data = update[node_name]
        print(f"[{node_name}] ", end="")
        if "queries" in node_data:
            print(f"generated {len(node_data['queries'])} queries")
        elif "findings" in node_data:
            print(f"found {len(node_data['findings'])} result(s)")
        elif "critique" in node_data:
            passed = node_data.get("critique_passed", False)
            print(f"critique: {'PASS' if passed else 'FAIL'} | repairs: {node_data.get('repair_count', 0)}")
        else:
            print("done")

    state = graph.get_state(config)
    if state.next == ("synthesiser",):
        print("\n=== HUMAN APPROVAL GATE ===")
        print(f"Findings: {len(state.values['findings'])} collected")
        print(f"Critique: {state.values['critique']}")
        for i, f in enumerate(state.values["findings"][:2]):
            print(f"  [{i+1}] {f[:120]}...")

        feedback = input("\nEnter feedback (Enter to approve): ").strip()
        if feedback:
            graph.update_state(config, {"human_feedback": feedback})

        print("\n--- Phase 2: Synthesis ---")
        for update in graph.stream(None, config=config, stream_mode="updates"):
            node_name = list(update.keys())[0]
            print(f"[{node_name}] done")

    final = graph.get_state(config)
    print("\n" + "="*60)
    print("FINAL REPORT")
    print("="*60)
    print(final.values.get("report", "(no report generated)"))
    print(f"\nThread ID: {thread_id}")
    return thread_id
```
