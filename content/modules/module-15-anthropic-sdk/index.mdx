# Anthropic Python SDK — Raw API, Full Control

**Framework Elective · Optional · Prerequisite: Module 06**

Most framework tutorials hide the API behind layers of abstraction. This module tears off the cover. You will use the raw `anthropic` Python package — no LangChain, no LangGraph, no wrappers — and build a production-grade tool-using agent from scratch.

By the end, every line of your agent does exactly what you wrote. No magic. No hidden retries you didn't ask for. No prompt templates you can't see.

## Why the raw SDK?

Every AI framework — LangGraph, CrewAI, AutoGen, LlamaIndex — is built on top of the same Messages API you are about to learn. When something breaks in a framework, the debugging trail leads here. When a framework lacks a feature, you extend it here. When you need to understand what a framework is actually doing, you read it here.

The raw SDK also has real advantages beyond pedagogy:

- **Zero overhead.** No middleware, no event bus, no state machine. Just Python.
- **Full observability.** Every token, every role, every content block is yours to inspect.
- **Perfect for simple pipelines.** A linear orchestrator-worker chain is 50 lines with the raw SDK, not 200 lines of graph configuration.
- **Foundation for framework work.** After this module, framework abstractions will feel transparent rather than opaque.

## What you will build

Over six lessons you will construct a **production research agent** that:

1. Accepts a user question
2. Decides which tools to call (web search stub, calculator, file writer)
3. Executes tools, feeds results back to the model, and loops until done
4. Streams output to the terminal token by token
5. Caches its system prompt so repeated calls are 60–90% cheaper
6. Tracks token cost per run and prints a summary
7. Retries rate-limit errors automatically with exponential backoff

All of this in pure Python with one dependency: `anthropic`.
