# Module 14 - Claude Code: Operating Agentic Coding Workflows

**Part 6 - Practitioner Tools** - 7 lessons - ~6 hours total

<LearningObjectives>
- Explain Claude Code as an agentic coding harness: model, tools, context, permissions, sessions, and verification
- Use plan mode, checkpoints, Git, and test commands to keep Claude Code work reviewable
- Configure `CLAUDE.md`, rules, auto memory, skills, MCP, and hooks without overloading context
- Choose permission modes and sandbox boundaries based on task risk
- Use subagents, agent view, worktrees, agent teams, and dynamic workflows for parallel work
- Explain Claude Managed Agents as a hosted agent harness with agents, environments, sessions, events, tools, and permission policies
- Complete a safe Claude Code workflow that ends in a small verified diff and durable project guidance
</LearningObjectives>

## What you'll build

You will configure Claude Code for a real repository, then use it to make a small verified improvement. The final lab asks you to create or refine `CLAUDE.md`, choose a permission mode, run a plan-first implementation, add one deterministic hook or permission rule, use a review pass, and decide whether agent view or subagents would help.

Then you will design a Managed Agents version of the same workflow: the agent configuration, environment, tool permissions, session events, success criteria, and data-handling caveats.
