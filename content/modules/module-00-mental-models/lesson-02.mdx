# Lesson 2 · Context Windows, Tokens, and Cost

The context window is the model's working memory. Everything the model "knows" during a single call is whatever you put in the context. It has no memory beyond that — unless you build one.

## What the context window is

When you call an LLM, you pass it a list of messages: system prompt, user messages, assistant responses, tool results. All of that becomes the context. The model reads the entire context on every call and produces the next response.

The context window has a maximum size measured in tokens. GPT-4o's context window is 128k tokens. Claude's is 200k. This means the entire conversation — everything the model can see — is bounded.

<Callout type="key">
The context window is the model's entire world. Anything not in the context doesn't exist for that call. This is why agents need memory systems — the context fills up, old information gets dropped, and the model forgets.
</Callout>

## Why context management matters for agents

A plain chatbot only needs to track a conversation. An agent accumulates much more:

- The original goal
- Every tool call it made
- Every tool result it received
- Intermediate reasoning steps
- Error messages and retries

Every one of these adds tokens. A simple 10-step agent might use 5,000–20,000 tokens just on tool results. If you're not managing the context, you'll hit the limit — or spend a lot more money than you expected.

## Tokens are the unit of cost

LLM providers charge per token — typically separately for input tokens (what you send) and output tokens (what comes back). Output tokens are usually 3–5× more expensive than input.

A rough mental model for pricing:
- 1,000 tokens ≈ 750 words
- A single agent turn with tools might use 2,000–10,000 tokens
- A 10-step agent run: 20,000–100,000 tokens
- At $5/million tokens input, that's $0.10–$0.50 per run

This sounds cheap per run, but at scale — or with a poorly designed agent that loops — costs compound fast.

```python
# Always log token usage during development
response = client.chat.completions.create(
    model="gpt-4o",
    messages=messages,
)
print(f"Tokens used: {response.usage.prompt_tokens} in, {response.usage.completion_tokens} out")
```

<Callout type="tip">
Add token logging to every agent call during development. You want to know your cost per run *before* you scale.
</Callout>

## The context window fills from the bottom

As an agent loops, new tool results and responses get appended to the message list. The oldest messages — including parts of your system prompt — can get pushed out if you hit the limit.

Some models handle this with a sliding window (dropping old messages). Others throw an error. Neither is what you want. The solution is deliberate context management: summarizing old tool results, keeping only relevant history, or using a memory layer.

You'll build this in Module 3. For now, the lesson is: your context is a resource. Design your agents like it costs money — because it does.
